# 智协库员工协作一期后端交接文档

## 1. 交接范围

本次交接对应“员工协作一期：组织关系与人员目录”。目标是让前端获得可靠的部门、岗位和直属汇报关系数据，为后续短期协作议题提供人员选择基础。

本次已完成：

- 用户组织字段扩展：部门 ID、直属上级、岗位、在职状态。
- 部门 Mock 数据和四个演示账号的上下级关系。
- `/organization` 查询接口。
- 前端 `OrganizationRepository` 和组织页面调用链。
- Prisma schema 的组织关系字段草案。

本次未完成：

- 管理员新增/编辑部门接口。
- 管理员调整员工部门、直属上级、岗位和状态的接口。
- Prisma Repository 和正式数据库 seed。
- WebSocket、即时聊天、协作议题和消息线程。

| 字段 | 类型 | 说明 |
|---|---|---|
| `departmentId` | string | 所属部门 ID |

## 5. 已实现接口

所有接口都需要：

```http
Authorization: Bearer <token>
```

响应继续使用：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 5.1 获取当前用户组织关系

```http
GET /api/organization/me
```

返回示例：

```json
{
  "employee": {
    "id": "u1",
    "name": "张晓宁",
    "account": "employee01",
    "departmentId": "d-rd",
    "departmentName": "产品研发部",
    "jobTitle": "产品经理",
    "managerId": "u2",
    "managerName": "李清越",
    "employmentStatus": "active"
  },
  "manager": {
    "id": "u2",
    "name": "李清越",
    "account": "manager01",
    "departmentId": "d-rd",
    "departmentName": "产品研发部",
    "jobTitle": "研发部负责人",
    "managerId": "",
    "managerName": "",
    "employmentStatus": "active"
  },
  "directReports": [],
  "departmentMembers": []
}
```

实际返回的 `departmentMembers` 会包含当前部门所有在职成员；`directReports` 只包含直接下属，不包含更深层级下属。

### 5.2 获取部门树

```http
GET /api/organization/departments/tree
```

返回部门树节点：

```json
{
  "id": "d-rd",
  "name": "产品研发部",
  "parentId": "",
  "leaderId": "u2",
  "sortOrder": 1,
  "status": "active",
  "leaderName": "李清越",
  "children": []
}
```

普通用户当前只返回本人所属部门；系统管理员返回全部启用部门。

### 5.3 搜索人员目录

```http
GET /api/organization/users?scope=team&keyword=张&departmentId=&activeOnly=true
```

查询参数：

| 参数 | 默认值 | 说明 |
|---|---|---|
| `scope` | `team` | `department`、`team`、`all`；只有系统管理员的 `all` 有效 |
| `keyword` | 空 | 匹配姓名、账号或岗位 |
| `departmentId` | 空 | 按部门过滤 |
| `activeOnly` | `true` | 是否仅返回在职人员 |

返回数组元素使用 `DirectoryItem` 结构，不返回密码、权限和收藏数据。

## 6. 权限和数据安全要求

当前查询规则：

1. 普通员工、审批人、知识管理员只能查看自己所在部门的在职成员。
2. 系统管理员使用 `scope=all` 时可以查看全部在职成员。
3. 人员目录不得返回 `password`、`permissions`、收藏列表等敏感字段。
4. 停用员工默认不出现在人员目录中。
5. 部门负责人和直属上级是两个独立概念，服务端不能通过 `role` 自动推导 `managerId`。

后续实现写接口时，必须在 Service 层校验：

- 员工和直属上级不能是同一个人。
- 直属上级必须存在且为在职员工。
- 部门必须存在且处于启用状态。
- 更新后汇报链不能出现循环。
- 非系统管理员不能修改组织关系。

## 7. Prisma 接入注意事项

`server/prisma/schema.prisma` 已增加：

- `Department` 模型。
- `User.departmentId`。
- `User.managerId` 自关联。
- `User.jobTitle`。
- `User.employmentStatus`。

但是当前仓库原有 schema 已使用多个 Prisma enum，而 SQLite connector 在当前 Prisma 版本下不支持这些 enum。执行：

```powershell
npm run prisma:generate
```

会在 schema 校验阶段失败。这不是本次组织字段新增造成的单一问题，而是原有数据库配置问题。正式接 Prisma 前，需要统一处理：

1. 将 SQLite 中的 enum 字段改为 `String` 并在 Service 层校验；或
2. 更换支持 enum 的数据库 connector；或
3. 重新整理 Prisma schema 后再生成客户端。

在该问题解决前，继续使用 `mock-db.ts` 完成接口和前端联调。

7. 组织查询接口不返回密码和权限字段。
