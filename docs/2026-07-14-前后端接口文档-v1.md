# 智协库前后端接口文档 v1

## 1. 文档说明

本文档用于支撑“智协库”项目的前后端解耦与联调，面向当前课程设计阶段的最小可运行闭环。

当前推荐技术方案：

- 后端框架：`NestJS`
- ORM：`Prisma`
- 数据库：`SQLite`
- 接口风格：`RESTful API`
- 数据格式：`JSON`

本文档优先覆盖以下功能：

1. 登录认证
2. 知识库查询
3. 知识收藏与纠错
4. 流程模板查询
5. 流程发起、审批、驳回、二次确认
6. 消息通知查询与已读
7. 助手问答

## 2. 基础约定

### 2.1 Base URL

本地开发环境建议：

```text
http://localhost:3000/api
```

### 2.2 认证方式

除登录接口外，其余接口默认使用 Token 鉴权。

请求头格式：

```http
Authorization: Bearer <token>
```

### 2.3 通用响应格式

成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

失败响应：

```json
{
  "code": 1001,
  "message": "参数错误",
  "data": null
}
```

### 2.4 通用错误码

| code | 含义 |
|---|---|
| 0 | 成功 |
| 1001 | 参数错误 |
| 1002 | 未登录或 Token 无效 |
| 1003 | 无权限访问 |
| 1004 | 资源不存在 |
| 1005 | 业务状态不允许当前操作 |
| 1006 | 数据库操作失败 |
| 1007 | 服务器内部错误 |

## 3. 枚举定义

### 3.1 用户角色 `UserRole`

| 值 | 含义 |
|---|---|
| employee | 普通员工 |
| approver | 审批人 |
| knowledgeAdmin | 知识管理员 |
| systemAdmin | 系统管理员 |

### 3.2 流程状态 `WorkflowStatus`

| 值 | 含义 |
|---|---|
| pending | 待审批 |
| in_review | 审批中 |
| pending_security_confirm | 待二次确认 |
| approved | 已通过 |
| rejected | 已驳回 |

说明：前端显示中文，后端接口建议统一使用英文枚举，避免后续扩展时状态文案和业务值混在一起。

### 3.3 审批决定 `ApprovalDecision`

| 值 | 含义 |
|---|---|
| submitted | 已提交 |
| approved | 已通过 |
| rejected | 已驳回 |
| security_confirmed | 已完成二次确认 |

### 3.4 通知目标类型 `NotificationTargetType`

| 值 | 含义 |
|---|---|
| workflow | 关联流程 |
| knowledge | 关联知识 |
| none | 无跳转目标 |

## 5. 接口列表

## 5.1 认证模块

### 5.1.1 登录

- 方法：`POST`
- 路径：`/auth/login`
- 描述：用户使用账号密码登录，返回 token 与用户信息

请求体：

```json
{
  "account": "employee01",
  "password": "123456"
}
```

成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "mock-jwt-token",
    "user": {
      "id": "u1",
      "name": "张晓宁",
      "account": "employee01",
      "department": "产品研发部",
      "role": "employee",
      "roleLabel": "普通员工",

      "favoriteKnowledgeIds": ["k2", "k4"],
      "recentKnowledgeIds": ["k1", "k2", "k3"],
      "todoCount": 2
```

### 5.3.5 发起流程

- 方法：`POST`
- 路径：`/workflow/instances`

请求体：

```json
{
  "templateId": "wf2",
  "title": "华南差旅报销",
  "extraInput": "差旅",
  "dateInput": "",
  "amountInput": 3680,
  "reasonInput": "华南客户拜访",
  "attachmentInput": "发票+行程截图"
}
```

说明：

- 不同模板需要的字段不同
- 首版为了快速落地，可保留通用表单结构
- 后续可以拆为不同模板 DTO

成功响应：

```json
{
  "code": 0,
  "message": "提交流程成功",
  "data": {
    "id": "i1001",
    "templateId": "wf2",
    "title": "报销申请 - 华南差旅报销",
    "status": "pending",
    "currentNode": "等待部门负责人审批",
    "requiresSecurityConfirm": false
  }
}
```

失败场景：

- 模板不存在
- 表单缺失
- 金额格式错误

### 5.3.6 审批通过

- 方法：`POST`
- 路径：`/workflow/instances/:id/approve`

请求体：

```json
{
  "comment": "材料齐全，建议通过。"
}
```

成功响应：

```json
{
  "code": 0,
  "message": "审批成功",
  "data": {
    "id": "i1",
    "status": "in_review",
    "currentNode": "等待系统管理员确认",

  }

```

说明：

- 流程是否进入下一节点
- 是否触发二次确认
- 是否直接结束

以上都由后端决定，前端只渲染返回结果。

## 6. 前后端联调建议

1. `POST /auth/login`
2. `GET /knowledge/categories`
3. `GET /knowledge/articles`
4. `GET /workflow/templates`
5. `POST /workflow/instances`
6. `GET /notifications`
7. `POST /notifications/:id/read`
