# A 同学鸿蒙特色能力设计

## 目标

在现有 `feature/a-knowledge-dashboard` 分支上，为智协库增加可演示且尽量使用真实 HarmonyOS API 的四项能力：桌面互动卡片、知识库图片 OCR、知识阅读字体调节、沉浸光感视觉效果。不得修改 B 同学负责的流程状态机、审批、安全和管理模块。

## 范围与约束

- 工程目标 SDK 为 HarmonyOS API 26，使用 ArkTS/ArkUI。
- OCR 入口只放在知识库页面，图片来源使用系统相册选择器，不申请相机权限。
- OCR 结果使用独立结果页展示图片、识别全文、关键词和相关知识；系统能力不可用或识别失败时使用明确的演示降级结果。
- 桌面卡片使用真实 `FormExtensionAbility`，通过本地持久化同步待办数、未读消息数和最近知识；卡片点击跳转到应用页面。
- 字体设置保留现有三档字号，增加标准/半粗字重，并在当前应用会话及本地设置中保持。
- 光感组件优先使用 `@kit.UIDesignKit` 的 `HdsVisualComponent`；设备不支持时回退到普通分层背景，不阻断业务页面。

## 方案

### 互动卡片

新增表单扩展能力、表单配置和卡片 ArkUI 页面。应用壳层将仪表盘摘要写入 Preferences，卡片扩展读取摘要并通过 `formProvider` 更新。卡片点击使用 `postCardAction` 打开 `EntryAbility` 并携带目标路由参数。卡片的业务数据只做摘要同步，不复制工作流状态机。

### OCR

知识库页面调用系统图片选择器得到图片 URI，解码为 `PixelMap` 后调用 `@kit.CoreVisionKit` 的 `textRecognition.recognizeText`。结果服务负责合并文本块、提取中文关键词并匹配现有知识列表。页面负责展示状态：选择中、识别中、成功、降级/失败。

### 字体与光感

阅读设置扩展为字号和字重两个值，由 `Index` 统一持有并传给 `KnowledgeDetailPage`。详情正文使用计算后的字号和 `FontWeight`。光感效果封装为可选组件，先检查系统能力或捕获 API 异常，再显示 HDS 效果或静态回退样式。

## 错误处理

- 相册取消选择：回到知识库，不显示错误弹窗。
- 图片解码/OCR 失败：保留结果页入口，显示演示识别文本并说明原因。
- 卡片数据读取失败：显示 0 待办、0 未读和默认知识标题。
- HDS 不支持或调用异常：显示静态背景，不影响点击和滚动。
- 所有系统 API 调用使用 `try/catch` 和日志，禁止因平台能力缺失导致应用启动失败。

## 验收标准

1. 编译通过，现有登录、工作台、知识库、详情、助手、消息、我的流程不回归。
2. 安装后可从桌面服务卡片入口添加智协库卡片，卡片显示摘要并可点击进入应用。
3. 知识库可从相册选图，识别成功显示文字和相关知识；不支持时有可解释降级结果。
4. 详情页字号和字重设置即时生效，返回后再次打开仍保持。
5. 支持 HDS 的设备显示光感效果，不支持设备显示静态回退样式。
6. Hypium 现有测试和新增服务/状态测试通过。
