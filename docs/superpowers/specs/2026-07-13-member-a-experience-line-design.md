# 智协库成员 A 体验线设计规格

## 1. 目标

在现有 HarmonyOS ArkTS / ArkUI 原型基础上，完成成员 A 负责的“知识库与工作台体验线”，形成从登录、工作台、知识检索、知识阅读、智能问答、消息处理到个人中心的完整可演示闭环。

实现必须保持成员 B 的流程协作、审批、权限治理、管理和安全能力边界，不重写 B 的流程状态机。A 与 B 通过明确的回调和公共类型完成联动。

## 2. 实现范围

成员 A 负责：

- 登录后的工作台体验。
- 企业知识库分类、搜索、列表、详情、收藏、有用反馈和纠错建议。
- 智能知识助手的模拟问答、关联知识、推荐流程和最近问答记录。
- 消息列表、未读状态和目标跳转。
- “我的”页面中的个人信息、收藏、消息和基础阅读设置。
- 互动卡片样式、知识阅读字体档位、沉浸式知识详情等可稳定演示的 HarmonyOS 展示点。

成员 A 不负责：

- 流程表单、审批节点、流程状态机和时间线。
- 用户、部门、角色、流程模板的管理功能。
- 高风险操作安全确认、审计日志和真实数字身份能力。
- 真实企业后端、真实大模型、真实 Agent / Skill 发布。

## 3. 设计原则

### 3.1 边界式拆分

保留 `Index.ets` 作为应用外壳、登录入口、底部导航和 A/B 共享状态装配点。将成员 A 的页面、通用组件和纯业务逻辑拆分到独立目录，减少两人在 `Index.ets` 上的并行修改。

### 3.2 保留现有视觉基线

继续使用现有蓝白办公风格，提升信息层次、交互反馈和移动端可读性，不进行影响 B 模块的大规模主题重构。

### 3.3 本地模拟但接口清晰

课程阶段使用内存模拟数据。页面不直接硬编码大段知识、消息或问答规则；数据通过服务函数提供，以便后续替换为 REST API。

### 3.4 不伪造平台能力

能够用 ArkUI 稳定实现的能力直接完成。依赖华为账号、签名、AGC、系统服务或特定真机的能力只保留接口和说明，不在页面中宣称已真实接入。

## 5. 导航设计

底部导航保持四项：

1. 工作台
2. 知识库
3. 流程
4. 我的

成员 A 的知识详情、智能助手和消息列表作为应用内部子页面打开。子页面复用统一页头并显示返回按钮，不增加新的底部导航项。

`Index.ets` 持有当前一级页签和当前子页面状态。A 页面通过回调请求打开知识详情、智能助手、消息或 B 的流程入口。

## 6. 页面设计

### 6.1 工作台

工作台包含：

- 当前用户姓名、部门和角色。
- 待办数量、知识数量、未读消息数量三项统计。
- 请假、报销、采购、权限申请快捷入口。
- 智能助手主入口。
- 最近浏览知识。
- 收藏知识摘要。
- 最近消息摘要。

统计卡片尺寸稳定，动态数字不会改变布局。快捷流程通过 `templateId` 回调 B 的流程入口，A 不创建流程实例。

### 6.2 知识库

知识库包含横向分类筛选、搜索输入、结果数量、知识卡片和空状态。搜索匹配标题、摘要、分类和标签，忽略关键词两端空格并进行不区分英文大小写的匹配。

普通用户只看到 `published` 状态的知识。`draft`、`deprecated` 和 `removed` 不进入普通搜索结果。

### 6.3 知识详情

详情页展示标题、分类、更新时间、版本、正文、标签、附件和关联流程，并提供：

- 收藏或取消收藏。
- “有用”反馈，单个用户对同一知识只记录一次。
- 纠错建议输入和提交。
- 三档阅读字体：小、标准、大。
- 关联流程入口；无关联流程时不展示发起按钮。

字体设置在当前应用会话内生效，并应用于知识正文。纠错内容去除首尾空格后不能为空。

### 6.4 智能知识助手

助手使用本地关键词规则返回：

- 答案摘要。
- 关联知识列表。
- 推荐流程。
- 问答时间。

空输入不调用匹配逻辑，也不写入历史。每次有效提问追加到最近问答记录，最新记录优先展示。问答结果可以打开知识详情或通过回调发起 B 的流程。

页面明确标识为“模拟智能助手”，避免将关键词规则描述为真实大模型或已发布系统级 Agent。

### 6.5 消息

消息页支持“全部”和“未读”两种筛选。用户可以打开消息并将其标记为已读，也可以显式点击“标记已读”。

消息需要包含目标类型和目标 ID：

- `knowledge`：打开知识详情。
- `workflow`：调用 B 提供的流程详情跳转回调。
- `none`：仅展示消息内容。

目标数据不存在时显示提示，不执行错误跳转。

### 6.6 我的

“我的”页面展示用户资料、角色权限、收藏数量、未读消息数量和基础设置入口。收藏知识可跳转详情，消息入口可打开消息页，阅读字体设置与知识详情共享。

退出登录后清理当前子页面和临时输入。重新登录时从对应模拟用户初始化收藏和最近知识。

## 7. 状态与数据模型

`Index.ets` 暂时持有以下共享状态：

- 当前用户。
- 当前一级页签和内部子页面。
- 当前选中知识 ID。
- 当前用户收藏知识 ID。
- 有用反馈记录。
- 纠错建议记录。
- 消息列表及已读状态。
- 智能助手输入、结果和历史。
- 阅读字体档位。

新增或扩展模型：

```ts
export type AppSubPage = 'none' | 'knowledgeDetail' | 'assistant' | 'messages';
export type ReadingFontScale = 'small' | 'standard' | 'large';
export type NotificationTargetType = 'knowledge' | 'workflow' | 'none';

export interface KnowledgeCorrection {
  id: string;
  articleId: string;
  userId: string;
  content: string;
  createTime: string;
}
```

`NotificationItem` 增加 `targetType` 和 `targetId`。知识文章不保存用户态收藏和反馈，避免把公共内容与个人状态混在一起。

## 8. A/B 集成契约

成员 A 从成员 B 使用：

```ts
onStartWorkflow(templateId: string): void
onOpenWorkflow(instanceId: string): void
todoCount: number
workflowTemplates: WorkflowTemplate[]
```

成员 B 从成员 A 使用：

```ts
getKnowledgeByWorkflowType(type: string): KnowledgeArticle[]
searchKnowledge(keyword: string): KnowledgeArticle[]
onOpenKnowledge(articleId: string): void
```

A 页面不能直接修改 `WorkflowInstance`、审批状态或审批记录。B 页面不能直接修改收藏、问答历史或纠错记录。

## 9. 异常与空状态

- 搜索无结果：显示空状态和清除筛选操作。
- 分类下无知识：显示该分类暂无已发布内容。
- 问答为空：提示输入问题，不创建历史记录。
- 问答无精确匹配：返回通用检索提示和有限的候选知识，不伪造确定答案。
- 纠错为空：提示补充具体问题。
- 消息目标不存在：提示内容暂不可用。
- 流程模板不存在：隐藏或禁用入口，不使用其他流程兜底。
- 未登录访问：由应用外壳返回登录页。

## 10. HarmonyOS 展示点

本阶段直接实现：

- ArkUI 工作台互动卡片式布局。
- 知识详情三档字体阅读体验。
- 沉浸式知识详情标题区和长文阅读布局。
- 智能助手的 Agent / Skill 模拟入口和清晰的模拟标识。
- 安全区域、滚动和移动端稳定尺寸适配。

视本机环境尝试：

- 使用 DevEco Studio 编译、运行模拟器并完成交互验证。
- 若工程和 SDK 支持，增加可运行的服务卡片原型。

需要用户环境配合的事项：

- 华为开发者账号登录和授权。
- 应用签名、证书或 AGC 项目绑定。
- 真实设备连接、USB 调试授权和设备端确认。
- 仅特定系统版本或设备支持的系统级 Agent、Skill、碰一碰、数字身份、数字盾和云调试能力。

## 11. 测试与验证

### 11.1 单元测试

为纯服务逻辑覆盖：

- 已发布知识过滤。
- 分类与关键词组合搜索。
- 问答关键词匹配和空输入。
- 消息全部/未读筛选。
- 消息标记已读。
- 纠错内容校验。

### 11.2 构建验证

使用本机 HarmonyOS 构建工具完成 debug 构建。构建失败时记录完整错误并修复到通过；若仅因签名或设备环境失败，必须区分代码编译问题与外部环境问题。

### 11.3 交互验收

至少验证以下路径：

1. `employee01` 登录并查看工作台。
2. 分类或搜索找到《差旅报销制度》。
3. 打开详情、调整字体、收藏、提交有用反馈和纠错建议。
4. 在助手中询问“报销打车费需要什么材料”。
5. 从回答打开知识详情并调用报销流程入口。
6. 打开消息、筛选未读并标记已读。
7. 在“我的”查看收藏和个人信息。
8. 切换到 `manager01`，确认用户态数据正确初始化且 B 的流程页仍可进入。

## 12. 验收标准

- AC-001：三种模拟角色均可登录，A 页面显示正确用户信息。
- AC-002：知识分类、搜索、详情和收藏链路完整。
- AC-003：智能助手返回相关知识和推荐流程。
- AC-007：消息可见、可读并能按目标跳转。
- AC-HM-001：工作台展示稳定的待办或消息互动卡片。
- AC-HM-002：智能助手体现模拟 Agent / Skill 思路。
- A 的新增单元测试通过。
- 工程 debug 构建通过，或只剩有证据证明的外部签名/设备阻塞。
- B 的流程页、发起入口和审批逻辑未被 A 的拆分破坏。
