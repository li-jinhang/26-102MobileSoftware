# 流程日期时间回显修复设计

## 问题

请假申请表单完成日期和时间选择后，开始时间或结束时间仍显示“请选择日期和时间”。确认回调已经更新 `workflowDateInput` 或 `workflowEndDateInput`，但 `WorkflowDateTimeSelector` 通过普通字符串参数读取旧值，界面没有随 `@State` 更新。

## 方案

保留现有日期弹窗、时间弹窗、结束时间晚于开始时间的校验及提交数据格式。调整 `WorkflowDateTimeSelector`，使它根据 `isEnd` 直接读取组件的 `workflowDateInput` 或 `workflowEndDateInput` 状态，并据此显示文字和颜色。

选择完成后，字段立即显示 `YYYY-MM-DD HH:mm`。开始时间变化且晚于已有结束时间时，继续按现有逻辑清空结束时间。

## 范围

仅修改请假表单日期时间选择器的显示绑定。不修改其他表单、后端接口、流程提交逻辑或视觉布局。

## 验证

1. 项目编译通过。
2. 选择开始时间后，开始时间字段立即回显。
3. 选择结束时间后，结束时间字段立即回显。
4. 结束时间不晚于开始时间时，仍提示错误且不写入字段。
5. 提交时仍使用 `开始时间 至 结束时间` 格式。
