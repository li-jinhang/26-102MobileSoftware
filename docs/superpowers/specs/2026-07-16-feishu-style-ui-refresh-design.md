# 飞书式企业协作界面改造设计

## 目标

在不改变现有业务逻辑、接口调用和导航关系的前提下，将鸿蒙应用改造成更接近飞书的企业协作界面：内容优先、层级清晰、信息紧凑、交互克制，并保持当前智能助手、知识库、流程、签到、消息和组织功能可用。

本轮不修改应用名称、桌面图标、`bundleName`、后端接口或未完成功能的内部逻辑。

## 视觉原则

- 页面底色使用浅灰，主要内容面使用白色，依靠分区、细分隔线和留白建立层级。
- 主色使用协作蓝，青绿色只用于状态或现有沉浸光感点缀，不使用大面积渐变。
- 卡片圆角统一控制在 6-8vp；减少阴影、悬浮大卡和卡片嵌套。
- 使用 4/8vp 间距体系，页面横向边距统一为 16vp，内容区垂直间距以 8/12/16/24vp 为主。
- 标题字号克制：页面标题 22-24fp，分区标题 16-18fp，正文 14-16fp，辅助信息 11-13fp。
- 主要点击区域至少 44vp，高频入口提供按压反馈；状态不能只通过颜色表达。

## 页面结构

### 应用外壳

- 顶部栏改成简洁标题栏：左侧显示当前页面，右侧保留消息入口与未读提示。
- 底部导航使用统一的线性符号与文字，不再使用“工/知/流/我”圆形占位标识。
- 选中态使用蓝色图标与文字，并以轻量背景或顶部指示增强辨识；未选中态使用中性灰。

### 工作台

- 顶部保留问候和沉浸光感，但降低装饰占比，使其成为轻量状态横幅。
- 待办、消息、考勤和邮件改成紧凑数据栏，支持快速扫描。
- 高频功能入口使用规则网格，统一图标容器与标题长度。
- 最近知识、收藏知识等内容使用列表式信息单元，减少整块大卡。

### 知识库

- 保留分类浏览，不恢复已迁移的搜索框和图片搜索按钮。
- 分类使用紧凑横向筛选或分段控件；知识条目使用标题、摘要、分类、更新时间的列表结构。
- 收藏状态作为次级标签，不抢占主标题视觉层级。

### 流程

- 总览、发起、我的流程使用分段控件切换。
- 待办和流程实例以状态清晰的列表展示，主操作靠近对应内容。
- 表单字段、日期时间选择器和按钮统一输入高度、边框与错误反馈样式。

### 我的

- 个人信息使用单一资料头部，不放在多层卡片中。
- 组织、消息、邮件、阅读设置和退出等入口采用标准设置列表，使用分隔线组织。
- 危险操作与普通入口保持明确的颜色和间距区分。

### 子页面

- 智能助手、消息、签到、邮件、知识详情和 OCR 结果统一顶部返回栏、页面背景、列表单元和按钮样式。
- 保留助手文字/图片输入、大模型处理、知识结果和表单跳转/自动填充逻辑。

## 组件与代码边界

- 新增语义化主题令牌，统一颜色、间距、圆角和常用尺寸。
- 调整 `SectionBlock`、`KnowledgeCard`、`EmptyState` 等公共组件，页面不再重复定义同类视觉数值。
- 只在现有页面组件内调整布局与样式，不改 use case、repository、service 和模型契约。
- 不引入第三方 UI 库，使用 ArkUI 原生组件与系统符号能力；若目标 SDK 对符号兼容性不足，则使用稳定的文本/几何后备方案。

## 异常与兼容

- 空状态、加载状态、禁用状态和错误提示保持可见，并统一语义颜色。
- 小屏设备允许文字换行，固定格式控件使用稳定尺寸，避免动态内容撑开导航和快捷入口。
- 深色资源暂不扩展完整深色主题，但所有新增颜色集中管理，为后续适配保留入口。

## 验收标准

- 登录后四个主导航页面视觉一致，页面之间不存在明显的圆角、颜色和间距跳变。
- 工作台首屏可看到核心状态和高频功能，信息密度高于当前版本。
- 知识库仍可分类浏览，智能助手仍可使用文字与相册图片分析。
- 流程表单、签到、消息、邮件、组织和详情页导航与提交行为不发生回归。
- 主 HAP 构建成功；用户在 Pura90 Pro 26 模拟器完成主要流程回归。
