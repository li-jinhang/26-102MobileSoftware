# 01_cover

各位老师好，我们是 TEAM17。今天汇报的项目是白泽，一款基于 HarmonyOS 的企业知识与流程协作应用。它的目标是把员工在移动端最常见的知识查询、智能问答、流程办理和协作跟进放到一个入口中。

# 02_closed_loop

白泽的核心不在于堆叠功能，而在于形成连续闭环。员工可以先查制度，再向助手提问，由系统推荐流程并智能预填，最后提交和跟踪处理结果。

# 03_architecture

在鸿蒙端，我们使用 ArkTS 和 ArkUI 组织组件化页面，并通过 UseCase、Repository 和 HTTP Adapter 分层。这样页面不直接绑定数据来源，可以从本地 Mock 平滑切换到后端接口。

# 04_brand_design

邢思琦负责名称与图标设计。白泽来自中国传统文化中辨识万物、通晓知识的意象，与项目的知识理解、流程推荐和风险提示定位相呼应。

# 05_internal_mail

内部邮件除了收件箱和写邮件能力，还能关联已有知识和业务流程。用户可以继续编辑助手生成的邮件草稿，使沟通信息带有完整业务上下文。

# 06_attendance

考勤页把今日签到签退状态、时间地点和备注集中呈现，并保留近期记录。这样既保证移动操作有明确反馈，也方便用户回访核对。

# 07_favorite_knowledge

知识详情支持收藏、纠错和关联流程。收藏后，员工能在个人中心回访高频制度，而不是重新检索，从阅读自然进入下一步办理。

# 08_backend

李金航负责后端开发。后端以 NestJS 按认证、知识、流程、通知、考勤、邮件、助手和协作划分模块，使用 Prisma 映射 SQLite，并以 Token 和统一响应结构支持前后端联调。

# 09_collaboration

协作模块首先呈现组织关系，帮助员工明确协作对象；随后围绕会议、任务等工作对象展示参与人、截止时间与处理状态，让协作事项可持续跟进。

# 10_smart_assistant

李承岩负责智能助手。用户既可以直接输入制度问题，也可以从相册选择图片。助手返回制度摘要、关联知识、推荐流程和缺失字段提示，避免用户在多个页面之间反复查找。

# 11_templates_knowledge

模板与知识库为不同业务场景提供标准入口。知识模块支持分类、详情、收藏和纠错；流程模板覆盖请假、报销、采购和权限等常见事项。

# 12_workbench

工作栏以当前身份、部门、待办数量和快捷入口为核心。不同角色登录后看到不同的待办和能力，用户可以快速到达知识、助手、流程和协作页面。

# 13_innovation

本项目最有代表性的创新点是智能预填。它不是把用户简单跳转到表单，而是从问题中提取标题、日期、金额、事由和附件等字段，写入后续流程草稿，用户补充缺失信息后即可提交。

# 14_testing

我们在 DevEco Studio 与 HarmonyOS 模拟器中运行移动端，后端运行 NestJS 服务并使用 SQLite 演示数据。测试覆盖登录、知识、助手、流程、通知、考勤、邮件和协作等主要路径，同时关注图片清晰度、Token 有效性与缺失字段。

# 15_thanks

以上是白泽的课程设计汇报。项目让我们完成了 ArkTS、ArkUI、前后端分层与业务协同的综合实践，也验证了从知识查询到业务执行的移动端闭环。谢谢老师。
