# 《智协库：面向公司内部的知识库与流程协作鸿蒙移动应用》需求文档

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 项目名称 | 智协库 |
| 课题名称 | 面向公司内部的知识库与流程协作鸿蒙移动应用 |
| 项目类型 | 移动软件开发课程小组课程设计 |
| 目标平台 | HarmonyOS |
| 开发技术 | ArkTS、ArkUI、REST API、关系型数据库或本地模拟数据 |
| 文档版本 | V1.0 |

## 2. 项目背景

在公司内部协作场景中，员工经常需要同时处理“查资料”和“办流程”两类任务。例如，提交报销申请前需要查看报销制度，申请权限前需要确认权限范围，审批采购单时需要参考采购规范。传统办公系统通常将知识库、审批流程、通知待办分散在不同入口中，导致员工查找成本高、流程填写容易出错、审批依据不清晰。

本项目拟设计并实现一款面向公司内部的鸿蒙移动应用“智协库”，将企业知识库、流程协作、智能问答、移动待办和安全确认整合在一个应用中。系统通过知识库辅助流程办理，通过流程沉淀组织经验，并结合 HarmonyOS 的智能化、全场景、多窗、安全与设计能力，提升移动办公效率和课程设计展示效果。

## 3. 项目目标

本项目的总体目标是构建一个适用于公司内部的移动端知识库与流程协作系统，帮助员工快速查询知识、发起流程、处理待办，并在关键业务场景中体现鸿蒙移动应用的特色能力。

具体目标如下：

1. 实现公司内部知识库的分类浏览、搜索、详情查看、收藏和反馈。
2. 实现请假、报销、采购、权限申请等典型流程的发起、审批、流转和历史查询。
3. 建立知识与流程之间的关联关系，使流程办理时能够自动推荐相关制度和历史知识。
4. 提供智能问答入口，支持员工用自然语言查询知识或跳转流程。
5. 结合 HarmonyOS 新能力规划互动卡片、闪控窗、碰一碰精准分享、数字身份和安全确认等特色功能。
6. 形成一个功能闭环完整、适合课程演示和答辩展示的移动软件课程设计作品。

## 4. 用户角色

| 角色 | 说明 | 主要权限 |
|---|---|---|
| 普通员工 | 公司内部普通使用者 | 查询知识、收藏知识、发起流程、查看自己流程进度 |
| 审批人 | 部门负责人或指定审批人员 | 查看待办、审批流程、填写审批意见、查看关联知识 |
| 知识管理员 | 负责维护企业知识内容 | 新增、编辑、发布、下架知识条目 |
| 系统管理员 | 负责系统基础配置 | 管理用户、部门、角色、流程模板和权限 |

### 5.1 范围内功能

- 首页工作台
- 企业知识库

- 完整企业级流程引擎
- 与真实 OA、ERP、HR 系统对接
- 大规模文档全文检索集群
- 真实电子签章平台对接
- 真实企业统一身份认证平台对接
- 复杂 AI 大模型私有化部署

上述能力可作为后续扩展方向。

## 6. 总体功能结构

系统主要分为四个一级模块：

| 一级模块 | 功能说明 |
|---|---|
| 工作台 | 展示待办、常用流程、最近知识、快捷入口和消息 |
| 知识库 | 提供企业知识分类、搜索、详情、收藏、反馈和管理 |
| 流程 | 提供流程发起、审批、进度跟踪、历史记录和关联知识 |
| 我的 | 提供个人信息、角色权限、收藏、消息、设置和安全记录 |

建议底部导航栏设置为：

- 工作台
- 知识库
- 流程
- 我的

### 7.3 企业知识库模块

#### 7.3.1 功能描述

知识库用于存储公司内部制度、规范、FAQ、项目资料、培训材料和操作指南。员工可以按分类浏览，也可以通过关键词搜索知识。

#### 7.3.2 知识分类

建议预设以下分类：

| 分类 | 示例内容 |
|---|---|
| 行政人事 | 请假制度、考勤制度、入职流程 |
| 财务报销 | 差旅报销、发票规范、费用标准 |
| 采购管理 | 采购申请、供应商选择、合同审批 |
| IT 支持 | 账号权限、设备申请、系统故障处理 |
| 项目资料 | 项目规范、开发文档、会议纪要 |
| 培训材料 | 新员工培训、部门培训、技术分享 |

#### 7.3.3 功能需求

| 编号 | 需求 |
|---|---|
| FR-KB-001 | 用户可以按分类浏览知识条目 |
| FR-KB-002 | 用户可以通过关键词搜索知识 |
| FR-KB-003 | 搜索结果应展示标题、摘要、分类、标签和更新时间 |
| FR-KB-004 | 用户可以查看知识详情 |
| FR-KB-005 | 知识详情页应展示标题、正文、附件、标签、版本和适用流程 |
| FR-KB-006 | 用户可以收藏知识条目 |
| FR-KB-007 | 用户可以对知识条目进行“有用”反馈 |
| FR-KB-008 | 用户可以提交纠错建议 |
| FR-KB-009 | 知识管理员可以新增、编辑、发布和下架知识 |
| FR-KB-010 | 已下架或已废止知识不应出现在普通搜索结果中 |

### 7.4 智能知识助手模块

#### 7.4.1 功能描述

智能知识助手提供对话式入口，员工可以用自然语言询问制度、流程和操作问题。系统根据知识库内容返回答案摘要，并推荐相关流程入口。

#### 7.4.2 示例场景

用户输入：

> 报销打车费需要什么材料？

系统返回：

> 根据《差旅报销制度》，打车费报销需要上传发票、行程截图和事由说明。你可以直接发起“报销申请”。

系统同时展示：

- 相关知识：《差旅报销制度》
- 推荐流程：报销申请
- 快捷按钮：立即发起

#### 7.4.3 功能需求

| 编号 | 需求 |
|---|---|
| FR-AI-001 | 用户可以在智能问答页面输入自然语言问题 |
| FR-AI-002 | 系统根据关键词、标签或知识分类匹配相关知识 |
| FR-AI-003 | 系统返回答案摘要和相关知识条目 |
| FR-AI-004 | 系统可以根据问题推荐相关流程 |
| FR-AI-005 | 用户可以从问答结果直接跳转到知识详情或流程发起页面 |
| FR-AI-006 | 系统保留最近问答记录 |

#### 7.4.4 鸿蒙能力结合

该模块可规划接入 HarmonyOS 的 Agent 和 Skill 能力：

- 通过 Agent 构建企业智能助理。
- 通过 Skill 将“查制度”“发起流程”“查看待办”等能力暴露给系统级智能入口。

课程实现阶段可以先采用关键词匹配和模拟回复方式实现。

### 7.5 流程协作模块

#### 7.5.1 功能描述

流程协作模块用于处理公司内部常见审批流程。用户可以发起流程，审批人可以查看待办并处理审批，所有流程状态通过时间线展示。

#### 7.5.2 流程类型

课程设计建议实现以下流程：

| 流程 | 发起人 | 审批人 | 说明 |
|---|---|---|---|
| 请假申请 | 普通员工 | 部门负责人 | 填写请假类型、时间、原因 |
| 报销申请 | 普通员工 | 部门负责人、财务 | 填写费用类型、金额、附件 |
| 采购申请 | 普通员工 | 部门负责人、采购负责人 | 填写物品、预算、用途 |
| 权限申请 | 普通员工 | 部门负责人、系统管理员 | 申请系统账号或访问权限 |

#### 7.5.3 功能需求

| 编号 | 需求 |
|---|---|
| FR-WF-001 | 用户可以查看可发起的流程模板 |
| FR-WF-002 | 用户可以填写流程表单并提交 |
| FR-WF-003 | 用户可以上传或添加附件 |
| FR-WF-004 | 系统提交流程后生成流程实例 |
| FR-WF-005 | 系统根据流程模板确定审批节点 |
| FR-WF-006 | 审批人可以查看待办流程 |
| FR-WF-007 | 审批人可以选择通过或驳回 |
| FR-WF-008 | 审批人必须填写审批意见或驳回原因 |
| FR-WF-009 | 用户可以查看流程时间线 |
| FR-WF-010 | 用户可以查看自己发起的历史流程 |
| FR-WF-011 | 系统在流程详情页展示关联知识 |
| FR-WF-012 | 流程状态变化时生成通知消息 |

#### 7.5.4 流程状态

| 状态 | 说明 |
|---|---|
| 草稿 | 用户填写但尚未提交 |
| 待审批 | 已提交，等待审批人处理 |
| 审批中 | 存在多个审批节点，流程正在流转 |
| 已驳回 | 审批人驳回申请 |
| 已通过 | 所有审批节点通过 |
| 已归档 | 流程结束并归档 |

## 8. 鸿蒙特色能力规划

本项目结合 HarmonyOS 7 新能力进行功能规划，使课程设计体现鸿蒙平台特色。

| 鸿蒙能力 | 项目应用方式 | 课程实现建议 |
|---|---|---|
| Skill | 将查制度、发起报销、查看待办封装为系统级快捷能力 | 可在文档和演示中规划，代码中预留接口 |
| Agent | 构建企业智能助理，支持知识问答和流程推荐 | 使用模拟智能问答实现基础效果 |
| 视觉 AI | 识别票据、截图、附件内容，辅助填写表单或打标签 | 可先用图片上传和模拟识别结果展示 |
| 碰一碰·精准分享 | 将电脑或平板上的图片、文档精准插入知识文章或流程附件 | 作为特色场景演示设计 |
| 互动卡片 | 在桌面展示待办数量、流程进度和常用知识入口 | 可实现卡片样式页面或元服务卡片原型 |
| 闪控窗 | 审批倒计时、流程状态以悬浮窗形式展示 | 可做悬浮状态组件模拟 |
| 分布式数字身份 | 用于企业员工可信身份认证 | 登录和关键操作中体现身份校验设计 |
| 数字盾 | 用于高金额报销、采购审批、电子签章等关键确认 | 实现“安全确认”弹窗和日志记录 |
| 星盾机密风控引擎 | 用于高风险操作前的设备风险判断 | 课程阶段模拟风险检测结果 |
| 沉浸光感组件 | 提升工作台、审批卡片和知识详情的视觉层次 | 在界面设计中使用沉浸式卡片和光感风格 |
| 可变字体 | 优化制度长文档阅读体验 | 在知识详情页体现字体大小和字重调节 |
| 云调试 | 通过 AGC 远程真机验证兼容性 | 作为测试方案写入课程报告 |

## 12. 典型业务流程

### 12.1 报销申请流程

1. 员工进入工作台，点击“报销申请”。
2. 系统展示《差旅报销制度》等关联知识。
3. 员工填写费用类型、金额、事由并上传附件。
4. 系统进行表单校验。
5. 员工提交申请。
6. 部门负责人收到待办通知。
7. 部门负责人查看报销详情和关联制度。
8. 部门负责人审批通过后流转到财务。
9. 财务审批完成后，流程状态变为已通过。
10. 系统通知员工流程结果。

### 12.2 知识查询到流程发起

1. 员工进入智能问答页面。
2. 员工输入“申请系统权限怎么做？”
3. 系统返回《系统权限申请规范》摘要。
4. 系统推荐“权限申请”流程。
5. 员工点击“立即发起”。
6. 系统跳转至权限申请表单，并预填部分信息。

### 12.3 知识纠错流程

1. 员工查看知识详情。
2. 员工发现内容过期，点击“纠错反馈”。
3. 员工填写反馈说明。
4. 知识管理员收到消息。
5. 知识管理员编辑并发布新版本。
6. 收藏该知识的用户收到更新通知。

## 18. 风险与应对

| 风险 | 影响 | 应对措施 |
|---|---|---|
| 鸿蒙新能力实际接入难度较高 | 影响特色功能实现 | 采用“真实页面 + 模拟能力 + 接口预留”的方式完成课程展示 |
| 流程引擎开发复杂 | 影响开发进度 | 使用固定流程模板和简单状态机实现 |
| 智能问答效果不稳定 | 影响演示效果 | 采用关键词匹配和预设问答库 |
| 小组分工不均 | 影响整体完成度 | 按模块拆分，统一接口格式 |
| 数据量不足 | 影响演示真实感 | 准备制度、流程、消息等模拟数据 |

## 19. 后续扩展方向

1. 接入真实企业统一身份认证。
2. 接入真实电子签章和数字盾能力。
3. 引入大模型和向量检索，提升知识问答准确率。
4. 支持多端协同，在手机、平板、电脑之间流转流程和附件。
5. 支持知识生命周期管理，包括审核、版本对比和失效提醒。
6. 支持流程数据统计分析，例如平均审批时长、部门流程量、常见驳回原因。

## 20. 总结

“智协库”面向公司内部移动办公场景，将知识库、流程协作和智能助手结合起来，解决员工查制度难、办流程慢、审批依据不统一的问题。项目既能体现移动软件开发课程中的页面设计、数据建模、状态流转、权限控制和接口调用能力，也能结合 HarmonyOS 7 的智能化、全场景、多窗和安全能力，形成具有平台特色的课程设计作品。
